### 메세지 추적
-----
1. 웹 요청이 클라이언트에서 서버로 향하는 도중 둘 이상의 프록시를 지나게 되는 경우가 존재
<div align="center">
<img src="https://github.com/user-attachments/assets/13f19774-0715-4746-affe-cb71d4541a88">
</div>

   - 예를 들어, 많은 회사들이 보안과 비용 절감을 위해 인터넷 접속 시 캐시 프록시 서버를 사용해, 많은 대형 ISP들이 성능 개선과 기능 구현을 위해 프록시 캐시 사용
   - 오늘날 웹 요청의 상당수가 프록시를 지나며, 동시에 성능상 이유로 세계 곳곳에 흩어져 있는 대리 캐시 저장고에 콘텐츠를 복제해두는 방식이 점점 더 흔해지고 있음
   - 프록시는 여러 벤더에 의해 개발되며, 서로 다른 기능과 버그들을 갖고 있으며 여러 조직에 의해 관리

2. 프록시가 점점 흔해지면서, 서로 다른 스위치와 라우터를 넘나드는 IP 패킷의 흐름을 추적하는 것 못지 않게, 프록시를 넘나드는 메세지의 흐름을 추적하고 문제점을 찾아내는 것도 필요한 일이 됨
