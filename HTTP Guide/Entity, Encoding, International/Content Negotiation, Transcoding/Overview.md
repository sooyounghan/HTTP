-----
### 내용 협상과 트랜스코딩
-----
1. HTTP는 클라이언트와 서버가 내용 협상(Content-Negotiation) 방법을 제공
   - 이 방법을 통해 하나의 URL이 여러 가지 리소스 중 적합한 것에 대응되도록 할 수 있음 (예) 같은 웹 페이지의 프랑스어와 영어 버전)
   - 여기서 서로 다른 버전을 Variant(배리언트)라고 부름

2. 서버는 특정 URL에 대해 어떤 콘텐츠가 클라이언트에게 보내주기 가장 적절한지 다른 판단도 할 수 있어야 함
   - 어떤 경우 서버가 커스터마이징 된 페이지를 자동으로 생성하기도 함
   - 예를 들면, 어떤 서버는 HTML 페이지를 WAP 환경 하에서 동작하는 WAL 페이지로 변환할 수 있음
   - 트랜스코딩은 HTTP 클라이언트와 서버 사이 내용 협상에 대한 응답에서 수행
